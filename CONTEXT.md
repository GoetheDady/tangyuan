# 汤圆

汤圆是一个面向大语言模型对话的桌面应用。本文记录产品和领域语言，帮助界面、运行配置和会话体验保持一致。

## Language

**Agent**:
独立 AI 实体，拥有自己的身份设定、技能、工作空间、会话集合和模型选择。同一用户可以拥有多个 Agent；所有 Agent 共享同一份用户信息和按模型服务保存的凭据。
_Avoid_: 助手实例、模型、聊天角色

**默认 Agent**:
应用内置且不能归档的“汤圆”Agent，是用户配置模型服务凭据和创建其他 Agent 的唯一入口。
_Avoid_: 管理员 Agent、系统 Agent、主 Agent

**用户资料**:
描述当前唯一用户的长期偏好、工作方式和边界，并由所有 Agent 共享的信息。
_Avoid_: Agent 用户信息、用户账号、每 Agent 用户画像

**Agent 创建**:
用户通过默认 Agent 的对话描述需求，由默认 Agent 调用受控工具生成新 Agent 和初始身份设定的过程。创建完成的新 Agent 可以立即开始普通会话。
_Avoid_: 创建表单、手动建目录

**Agent 归档**:
将 Agent 从日常使用列表中移除，但保留其身份、会话和工作空间的可恢复状态。
_Avoid_: 删除 Agent、永久删除

**工作空间**:
每个 Agent 独立拥有的本地文件区域，用于保存该 Agent 工作时读取或产出的文件。MVP 中一个 Agent 只有一个工作空间，不与其他 Agent 共享。
_Avoid_: Agent Home、项目目录、共享目录

**共享 Skill**:
安装一次并对所有 Agent 可用的能力说明与配套资源。
_Avoid_: 全局插件、公共脚本

**Agent 专属 Skill**:
只对指定 Agent 可用的能力说明与配套资源；与共享 Skill 同名时，专属版本生效。
_Avoid_: 私有插件、Agent 脚本

**黑芝麻汤圆主题**:
汤圆的统一极简视觉语言，以糯米白、瓷白和黑芝麻墨黑为主体，并用少量朱砂红和状态色表达重点。它依靠留白、排版和对齐呈现安静、清晰的桌面工作感，不依靠装饰堆叠或餐饮意象。
_Avoid_: 米黄色主题、棕色餐饮风、纯黑白主题、渐变发光、玻璃拟态、卡片堆叠

**聊天主界面**:
用户与大语言模型进行连续对话的主要工作区。它只承载会话列表、消息流、输入与运行中的对话动作。
_Avoid_: 工作台、控制台、运行时面板

**设置页面**:
与聊天主界面和初始化配置页面分离的管理页面，用于维护共享模型服务凭据、各 Agent 的模型选择、状态、Skills 和归档。
_Avoid_: 控制台页面、运行时配置页面、设置弹窗、调试面板

**初始化配置页面**:
模型服务尚未满足启动对话条件时，用户配置 Provider、凭据和默认 Agent 初始模型的独立页面。
_Avoid_: 设置页面、欢迎页、身份初始化页面

**配置阻断**:
因模型服务、模型或凭据尚未满足启动对话条件，应用临时阻止用户进入聊天主界面并导航到初始化配置页面的状态。配置完成后应回到用户原本要继续的对话目标；没有明确目标时进入新对话。
_Avoid_: 错误页、运行时控制态

**初始化阻断**:
默认 Agent 的 `bootstrap.md` 仍存在时，汤圆持续补问初始化信息并暂不处理普通任务的状态。该状态不绑定某个特殊会话，删除 `bootstrap.md` 后结束。
_Avoid_: bootstrap session、首次会话、身份初始化页面

**配置入口**:
用户从聊天主界面进入设置页面的导航入口。它只负责页面导航，不在聊天主界面直接展开模型或凭据控件。
_Avoid_: 运行时控制、接口密钥按钮

**对话消息**:
用户发送的内容和大语言模型面向用户的回复内容。聊天主界面的消息流以对话消息为主，并可以展示与 Agent 回复相关联的执行历史；执行历史本身不属于对话消息。
_Avoid_: transcript 全量消息、活动消息、系统消息

**Agent 执行历史**:
Agent 为形成一次回复而产生并向用户公开的过程记录，按回合分段、回合内再按步骤展开，包括思考内容、中间文字、工具调用及其结果。执行历史与对应的 Agent 回复相关联，使用户能够查看任务如何推进，但不等同于会话中的对话消息。思考步骤保留 Runtime 提供的原文，不改写为摘要；工具步骤以用户可理解的摘要表达，不把完整工具参数或原始输出作为执行历史正文。
_Avoid_: 系统消息、调试日志、原始工具日志、transcript 全量消息

**上下文压缩提示**:
会话自动压缩上下文时，在消息流对应位置展示的轻量系统提示。它帮助用户理解上下文边界的变化，但不属于对话消息或 Agent 执行历史。
_Avoid_: 压缩消息、Agent 回复、执行步骤

**对话动作**:
Agent 运行期间需要用户处理后才能继续的交互，包括命令审批和问题澄清。一次澄清只提出一个问题；需要继续确认时，由 Agent 依次提出后续问题。等待处理时，对话动作以可操作形式出现，并阻止用户发送新的普通消息；处理完成后失去交互能力，并作为 Agent 执行历史的一部分保留用户的选择结果。
_Avoid_: 对话消息、设置表单、一次性弹窗

**执行尝试**:
Agent 针对一条用户消息进行的一次完整处理过程，具有独立的执行历史和完成、失败或中断结果。同一条用户消息可以在失败后发起新的执行尝试；新的尝试不重复产生用户消息，先前失败的尝试仍可回看。多次执行尝试分别展示，不把各次历史或结果合并。一次执行尝试由一个或多个回合组成。
_Avoid_: 重复消息、续写、覆盖重试

**回合**:
一次执行尝试内的一次完整大模型生成循环：模型产出一段回应（可能含思考与面向用户的文字），并执行由这段回应触发的一批工具调用。模型拿到工具结果后继续生成就进入下一回合；最后一个回合通常不含工具，只给出最终答复。回合边界与序号由 Runtime 依据底层 SDK 给出，不由上层自行推断。
_Avoid_: 轮次、生成轮、工具循环、LLM turn

**步骤**:
一个回合内、按 Agent 实际产生顺序排列的一个原子过程单元，分为思考步骤（模型推理原文）、文字步骤（面向用户的中间文字）和工具步骤（一次工具调用及其结果摘要）。同一回合内思考与文字可交错，均排在工具批次之前；工具步骤按启动顺序排列。
_Avoid_: 活动、事件、条目

**会话**:
用户与 Agent 进行一次连续对话的独立上下文，包含自身的消息历史和基本描述信息。
_Avoid_: 聊天记录、对话文件

**会话扩展数据**:
汤圆附加在现存会话上的产品专属信息。它不属于会话自身的内容或基本描述信息，也不能脱离所属会话独立存在。
_Avoid_: 会话元数据、孤立扩展数据
