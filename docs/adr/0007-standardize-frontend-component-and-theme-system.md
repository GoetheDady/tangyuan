# 统一前端组件与主题体系

Renderer 使用 shadcn/ui 源码组件、Radix UI 交互基础、Tailwind CSS 和 Lucide 图标，不混入 Ant Design、MUI 等第二套完整组件体系。所有界面图标统一来自 Lucide；不混用其他图标库，也不在已有图标可用时手绘 SVG。组件优先组合已有 shadcn/ui 能力，交互、无障碍和视觉状态因此保持一致。

Tailwind CSS 升级到实施时的最新稳定 v4，并由 `pnpm-lock.yaml` 锁定实际版本；后续升级必须显式执行，不使用浮动版本。开发组件前先通过 shadcn skill 和官方 CLI 查询现有组件、文档与差异，能组合 shadcn 组件时不重复手写；更新已安装组件时先预览 diff，保留本地修改。

全局样式只保留一套 shadcn 语义主题变量，底层使用 OKLCH 颜色；业务组件不得直接写颜色值或另建平行品牌变量。主题以“黑芝麻汤圆”为视觉设定：糯米白和瓷白承载背景与表面，黑芝麻墨黑承载文字和主要动作，冷灰承载边框与弱化状态，少量朱砂红作为品牌强调；成功、警告、信息和破坏状态分别使用玉绿、琥珀、青瓷蓝绿和深红。MVP 只提供这一套主题，不支持暗色模式或主题切换。

视觉风格保持极简：每个页面只有一个明确主操作，依靠留白、字号、字重和对齐建立层级，避免用大量容器表达结构。界面不使用渐变、玻璃拟态、发光效果、装饰性插画、夸张动效或大面积品牌色；阴影只用于确有层级关系的浮层，边框保持低对比度，圆角克制，不把所有内容包装成卡片或胶囊。

“黑芝麻汤圆”只作为配色和材质感来源，不把聊天页做成餐饮宣传界面。日常工作区域保持瓷白、墨黑和冷灰，朱砂红仅用于当前选择、关键提示和少量品牌焦点；信息密度由布局控制，不靠装饰填充空白。

界面动画统一使用 Motion，覆盖页面切换、组件进入、展开和运行状态反馈。MVP 不使用 GSAP；复杂时间轴动画不是工作型桌面界面的需求，避免两套动画运行时和两种动画写法。

短暂、非阻断反馈统一使用 shadcn 推荐的 Sonner toast；字段校验错误贴近字段展示，持续性错误使用页面内 Alert，删除 Agent 等破坏性动作使用 AlertDialog 二次确认。项目不自研第二套通知组件。
